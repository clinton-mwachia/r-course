Introduction

Machine learning is a powerful approach to extract insights and make predictions from data. In this lesson, we will explore how to implement machine learning using linear models in R. We will cover the entire process, from loading and cleaning the data to model training, evaluation, and making predictions.

The lesson aims to predict the sale price of a house using other features provided.

NOTE: THE STEPS PROVIDED HERE ARE NOT THE ONLY WAY OF DOING ML, YOU SHOULD TRY DIFFERENT APPROACHES AND THEN CHOOSE THE BEST FOR YOUR DATA.

# helper function to calculate na percentages of each columns
calc_na = function(data){
  
    na_val = colSums(is.na(data)) / nrow(data)
  
  return(na_val*100)
}

Exploratory Data Analysis

To better understand our data, we will perform data visualizations to answer the following questions: -

  1. Are old houses cheaper or more expensive?
  2. Does a large lot area increase the sale price?
  3. Which year has the highest average sale price?
  4. Houses with large garages are more expensive?

Let’s get started.

Are old houses cheaper or more expensive?

To answer this question, we will plot the year build of the house verses the sale price. This will us know the relationship between the year the house was built and the sale price.

imputed_data %>%
  select(SalePrice, YearBuilt) %>%
  arrange(YearBuilt) %>%
  ggplot(aes(x=YearBuilt, y=SalePrice)) +
    geom_point() +
    geom_smooth(method = "lm") +
    theme_bw() +
    ggtitle("Sale Price changes by Year of the house")
## `geom_smooth()` using formula = 'y ~ x'

CONCLUSION: HOUSES THAT ARE RECENTLY BUILT ARE MORE EXPENSIVE COMPARED TO THOSE THAT ARE BUILD A LONG TIME AGO.

Does a large lot area increase the sale price?

imputed_data %>%
  select(SalePrice, LotArea) %>%
  ggplot(aes(x=LotArea, y=SalePrice)) +
    geom_point() +
    geom_smooth(method = "lm") +
    theme_bw() +
    ggtitle("Sale Price changes by house lot Area")
## `geom_smooth()` using formula = 'y ~ x'

CONCLUSION: THE SALE PRICE OF A HOUSE INCREASES WITH THE LOT AREA. A HOUSE WITH A LARGE LOT AREA IS EXPENSIVE COMPARED TO A HOUSE WITH A SMALL LOT AREA.

Which year has the highest average sale price?

imputed_data %>%
  select(SalePrice, YrSold) %>%
  group_by(YrSold) %>%
  summarise(
    avg_salePrice = mean(SalePrice)
  ) %>%
  ggplot(aes(x=YrSold, y=avg_salePrice)) +
    geom_point() +
    geom_line() +
    theme_bw() +
    ylab("Average Sale Price") +
    ylab("Year Sold") +
    ggtitle("House Sale Price average by Year Sold")

CONCLUSION: HOUSES THAT WERE SOLD ON 2006 AND 2007, HAD THE HIGHEST AVERAGE SALE PRICES AS COMPARED TO THE OTHER YEARS.

The interpretation of what constitutes a “good” value for RMSE depends on the specific context and the scale of the target variable. In general, a lower RMSE indicates better model performance, as it signifies smaller differences between the predicted values and the actual values. However, there is no universally defined threshold for a “good” RMSE value that applies to all situations.

To assess the goodness of an RMSE value, it’s helpful to consider the scale and context of the problem you are addressing. You can compare the RMSE to the range of the target variable or the variability in the data. Additionally, it can be beneficial to compare the RMSE of different models or variations of the same model to determine which one performs better.

It’s important to note that the notion of a “good” RMSE value is relative and can vary depending on the specific domain, application, and the goals of your analysis. Therefore, it is recommended to interpret the RMSE in conjunction with other evaluation metrics and domain knowledge to make informed decisions about model performance.
